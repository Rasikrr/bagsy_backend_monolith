# План рефакторинга генерации доступных слотов (GetAvailableSlots)

Этот документ описывает стратегию реализации логики поиска свободного времени для записи с учетом типов расписания и интервальной арифметики.

## 1. Входные доменные данные

### Типы расписания (Location.ScheduleType)
1.  **Fixed (Фиксированное)**: Мастера работают строго по графику локации. Индивидуальные графики сотрудников игнорируются (или отсутствуют).
2.  **Mixed (Смешанное)**: У каждого мастера свой график, который должен находиться внутри интервала работы локации.

### Сущности расписания (Domain Schedule)
*   **Work Slot**: Интервал, в который разрешена запись.
*   **Rest Slot**: Технический перерыв, обед или блокировка времени, в которую запись запрещена (даже если это внутри Work Slot).

## 2. Уровень Use Case: Оптимизация данных

### Стратегия загрузки
*   **Адаптивность**: Если `Location.ScheduleType == Fixed`, Use Case **не запрашивает** `EmployeeScheduleSlots` из репозитория.
*   **Параллелизм**: Использование `errgroup` для одновременного получения:
    *   `LocationScheduleSlots`
    *   `EmployeeScheduleSlots` (только для Mixed)
    *   `Occupied Appointments`
    *   `Employee Profiles`
*   **Batching**: Запрос расписаний всех мастеров одним вызовом по списку ID.

## 3. Алгоритм генерации (slots.go)

Логика должна быть разделена на два сценария в зависимости от `ScheduleType`.

### Сценарий A: Fixed
1.  **База**: Взять `Work Slots` локации.
2.  **Вычитание**: Удалить интервалы `Rest Slots` локации.
3.  **Занятость**: Вычесть существующие `Appointments` конкретного мастера.
4.  **Фильтрация**: Удалить интервалы, начало которых меньше `time.Now()`.

### Сценарий B: Mixed
1.  **Пересечение (Work)**: Найти пересечение `Work Slots` локации и `Work Slots` мастера.
    *   *Rule*: `EffectiveWork = LocationWork ∩ EmployeeWork`.
2.  **Объединение (Rest)**: Найти объединение всех перерывов.
    *   *Rule*: `TotalRest = LocationRest ∪ EmployeeRest`.
3.  **Доступное время**: Вычесть перерывы из эффективной работы.
    *   *Rule*: `Available = EffectiveWork \ TotalRest`.
4.  **Занятость**: Вычесть `Appointments` мастера.
5.  **Фильтрация**: Удалить интервалы в прошлом.

## 4. Технические детали реализации

### Интервальная арифметика
*   Реализовать функции:
    *   `findIntersection(A, B)`: Пересечение двух наборов интервалов.
    *   `subtractIntervals(Base, Subtrahend)`: Вычитание одного набора из другого.
    *   `splitIntoSlots(Interval, Duration, Step)`: Нарезка интервала на доступные тайм-слоты.

### Валидация
*   Слот создается только если `EndAt - StartAt >= Service.Duration`.
*   Слоты генерируются с шагом `Location.SlotDurationMinutes`.
