# Алгоритм генерации временных слотов (GenerateSlots)

Этот документ описывает логику работы системы бронирования в Bagsy для расчета доступного времени (слотов) сотрудников и локаций.

## Обзор
Алгоритм преобразует сырые данные из базы данных (графики работы, существующие записи, перерывы) в упорядоченный список временных интервалов (`TimeSlot`), доступных для записи клиента.

---

## Ключевые этапы работы

### 1. Подготовка и группировка (Grouping)
Все входные данные (графики локаций, сотрудников и существующие записи) группируются по дате (`YYYY-MM-DD`). Это оптимизирует поиск и позволяет алгоритму эффективно обрабатывать большие диапазоны дат.

### 2. Расчет ежедневных интервалов (`calculateDailyIntervals`)
Для каждого дня в запрашиваемом периоде вычисляется "чистое" рабочее время:

1.  **Определение базового окна (Work Window):**
    *   **Fixed Schedule:** Используются только рабочие часы локации.
    *   **Mixed Schedule:** Вычисляется **пересечение** рабочих часов локации и сотрудника.
2.  **Вычитание перерывов (Rest):**
    *   Из рабочего окна удаляются технические перерывы локации и личные перерывы сотрудника.
3.  **Вычитание занятости (Occupied):**
    *   Удаляются интервалы времени, на которые уже созданы подтвержденные записи (`Appointments`).

### 3. Поддержка ночных смен (`mergeIntervals`) — **New**
Это критический этап для поддержки круглосуточной работы и ночных смен:
*   Алгоритм собирает все рассчитанные ежедневные интервалы в один общий список.
*   Функция `mergeIntervals` сортирует их по времени и **склеивает** те, что соприкасаются в полночь (`24:00` одного дня и `00:00` следующего).
*   **Результат:** Единый непрерывный таймлайн, игнорирующий границы календарных суток.

### 4. Нарезка слотов (Slicing)
На полученном таймлайне запускается цикл генерации конкретных предложений:
1.  Устанавливается курсор в начало доступного окна.
2.  Проверяется, помещается ли услуга (`serviceDuration`) от курсора до конца окна.
3.  Если да — создается `TimeSlot`.
4.  Курсор сдвигается вперед на шаг (`slotStep`, по умолчанию 30 мин).
5.  Процесс повторяется до конца окна.

*Благодаря склейке (этап 3), услуга может начаться в 23:30 и закончиться в 00:30 следующего дня.*

### 5. Валидация в реальном времени (`ValidateSlotAvailability`)
При попытке совершить бронирование или при выборе конкретного времени в интерфейсе, система выполняет строгую проверку выбранного интервала:

1.  **Определение временных границ:**
    *   Вычисляется время завершения услуги: `endAt = startAt + serviceDuration`.
    *   Определяются даты начала и окончания (если услуга начинается в 23:30 и длится 60 минут, даты будут разными).
2.  **Сбор контекста:**
    *   Запрашиваются графики и записи за все затронутые даты (от `dayStart` до `dayEnd`).
3.  **Построение таймлайна:**
    *   Аналогично генератору, для каждой даты рассчитываются доступные интервалы (`calculateDailyIntervals`).
    *   Все интервалы объединяются через `mergeIntervals` в единую линию времени.
4.  **Проверка вхождения:**
    *   Система ищет хотя бы один склеенный рабочий интервал, в который **полностью** помещается запрошенный отрезок `[startAt, endAt]`.
5.  **Проверка выравнивания (Alignment):**
    *   Дополнительно проверяется, соответствует ли `startAt` установленному шагу записи (`slotStep`) относительно начала рабочего интервала. Это предотвращает создание записей в произвольное время (например, в 10:07, если шаг — 30 минут).

---

## Ограничения и особенности

*   **Midnight Transition:** Для корректной работы ночной смены в БД должны быть созданы две записи (до 24:00 и после 00:00), так как база данных требует `start_time < end_time`.
*   **Buffer Times:** Текущая версия алгоритма не учитывает "техническое время" между записями (например, на уборку). Слот следующего клиента может начинаться ровно в момент окончания предыдущего.
*   **Past Filtering:** Алгоритм автоматически отфильтровывает любые слоты, время начала которых меньше текущего момента (`now`).

## Пример работы (Ночная смена)

**Входные данные:**
*   Смена: `10 марта 22:00 - 00:00` и `11 марта 00:00 - 06:00`.
*   Услуга: `60 минут`. Шаг: `30 минут`.

**Процесс:**
1.  Интервалы объединяются в один: `22:00 - 06:00`.
2.  Генерируется слот `23:00 - 00:00`.
3.  Генерируется слот `23:30 - 00:30` (**Успешно**, так как таймлайн не прерван).
4.  Генерируется слот `00:00 - 01:00`.
