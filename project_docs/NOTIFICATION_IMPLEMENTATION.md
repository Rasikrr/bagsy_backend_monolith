# Отчет по реализации уведомлений о записи

## Обзор
Была проведена ревизия и доработка системы уведомлений о записях (bagsy notifications). Основной целью было обеспечение отправки уведомлений как клиенту, так и мастеру, с поддержкой каналов WhatsApp и SMS (fallback).

## Выполненные работы

### 1. База данных
*   Создана миграция `migrations/20260123000000_add_recipient_to_notifications.sql`.
*   Добавлена колонка `recipient_type` (client, master) в таблицу `bagsy_notifications`.
*   Обновлен уникальный индекс: теперь уникальность определяется по `(bagsy_id, type, recipient_type)`, что позволяет создавать независимые уведомления одного типа для разных получателей.

### 2. Доменная логика
*   Добавлен enum `RecipientType` (`client`, `master`) в `internal/domain/notification/recipient.go`.
*   Обновлена структура `Notification` в `internal/domain/notification/notification.go`: добавлено поле `RecipientType`.
*   Сгенерирован код для нового enum (`go generate`).

### 3. Репозиторий
*   Обновлены модели и SQL-запросы в `internal/repositories/notifications/` для поддержки нового поля `recipient_type`.
*   Методы `Create` и `Upsert` теперь сохраняют тип получателя.

### 4. Сервис планирования (Bagsy Notifications Service)
*   Обновлена логика `buildNotifications` в `internal/services/bagsy_notifications/service.go`.
*   Теперь для каждого временного слота (за день, за час) создается **два** уведомления: одно для клиента, одно для мастера.

### 5. Обработка уведомлений (Job)
*   Обновлена джоба `processNotification` в `internal/jobs/bagsy_notifications/job.go`.
*   Добавлена логика выбора номера телефона в зависимости от `RecipientType`:
    *   Если `RecipientType == client` -> используется `b.ClientPhone`.
    *   Если `RecipientType == master` -> используется `b.MasterPhone`.
*   Тип получателя передается дальше в адаптер мессенджера.

### 6. Отправка уведомлений (Notification Service)
*   Обновлен интерфейс и реализация `SendBagsyReminder`.
*   Реализована разная логика формирования текста сообщения:
    *   **Клиент:** "Напоминаем о вашей записи..." (с указанием имени мастера).
    *   **Мастер:** "Напоминание: у вас запись..." (без указания своего имени, чтобы избежать тавтологии).

## Результат
Система теперь корректно планирует и отправляет персонализированные уведомления обеим сторонам (Мастеру и Клиенту) за день и за час до записи, используя WhatsApp с автоматическим переключением на SMS в случае ошибки.
